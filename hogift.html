1. 系統核心目標

透過 AI 輔助，實現企業禮品客製化流程的高效、智慧與精準。

對客戶 (前台): 提升體驗與效率

  • 提供即時、個人化的禮品推薦。
  • 提供視覺預覽 (打樣)，加速決策流程。

對營運 (後台): 賦能智慧管理與精準行銷

  • 自動化 產品標籤 、產品文案 與提案簡報 生成。
  • 智慧管理商品與客戶數據，可供後續生成受眾包或數據分析使用。

  • 禮品推薦: 透過自然語言對話進行精準推薦。
  • 圖像生成: 實時生成客製化禮品打樣預覽圖。

MongoDB存取商品主檔、會員資料

3. AI 禮品推薦流程 (前台流程 1)

透過自然語言對話，獲得 AI 提供的禮品推薦。

1.1 客戶發起對話 (HTTPS Request)

企業客戶輸入需求 (e.g., "我需要 100 份送給 IT 工程師的環保禮品")

1.2 取得禮品主檔 (MongoDB 查詢)

AI Agent 向 MongoDB 查詢所有商品主檔資料。

1.3 媒合與篩選 (BigQuery 查詢)

AI Agent 向 BigQuery 查詢預貼好的產品標籤，進行媒合 (提供AI向量查詢與全文檢索結果)。

1.4 反饋推薦 (HTTPS Response)

AI Agent 將經過排序與精選的推薦結果回傳至前台。

4. AI 禮品打樣流程 (前台流程 2)

客戶可上傳素材或提供創意，即時生成禮品打樣的預覽圖。

2.1 客戶提供素材/創意 (HTTPS Request)

客戶上傳 Logo 或輸入設計理念 (e.g., "將 Logo 以燙金印在杯子上")

2.2 生成預覽圖 (AI Agent 服務呼叫)

Giftpack AI 執行圖像生成 (Image Generation) 服務，將客製化效果應用於產品圖。

2.3 客戶確認與訂單成立 (MongoDB 寫入)

客戶確認打樣預覽圖，系統將最終訂單資料寫入 MongoDB。

核心服務：後台管理系統

使用者: 營運人員、客服、產品經理

組件名稱 職責

AI Agent (後台)執行貼標、文案生成、簡報內容生成等背景任務

6. AI 輔助營運功能

AI 產品文案生成

1.1 產品上架

營運人員提供產品圖片素材。

1.2 AI 生成文案 (AI Agent 呼叫)

AI 根據素材生成多版本產品文案。

1.3 文案回存 (MongoDB 寫入)

存入MongoDB(作為 AI 草稿或產品主檔)。

AI 提案簡報生成

2.1 取得訂單/需求資料 (MongoDB 查詢)

後台系統取得相關訂單或產品資料。

2.2 請求生成簡報內容 (AI Agent 呼叫)

系統將資料傳送給 AI Agent 請求生成文字結構。

2.3 Response 簡報 (HTTPS Response)

AI Agent 生成簡報內容並回傳至後台。

7. 自動化排程服務 (後台流程 3)

確保數據品質與行銷精準度。

排程 G1: 商品族群貼標排程

目標: 自動分類產品與貼上標籤 (e.g., #環保, #高階)。

  • 撈取MongoDB 中未貼標產品。
  • AI Agent 將貼標結果儲存至BigQuery。

排程 G2: 行銷受眾包媒合排程

目標: 建立產品與潛在客戶群的精準媒合包。

  • 取得產品 (Mongo) & 客戶資料 (BigQuery) 傳給AI Agent。
  • AI Agent 進行媒合，將受眾包結果回存BigQuery。

8. 核心技術組件

技術層 組件名稱 核心職責
應用服務層 Azure App Service 負責運行前台與後台服務，處理所有 Web 請求。
AI 輔助服務 Gemini LLM(AI Agent) 實現所有 AI 智能：圖像生成、自然語言推薦、貼標、文案與簡報生成。
資料層 MongoDB 結構化主檔資料： 儲存商品、會員、訂單、AI 文案草稿等。
資料層 BigQuery 分析型數據： 儲存 Log、AI 貼標結果、受眾包，用於複雜查詢。

開發工項

A. 企業客戶前台 (Client-Facing Web App)

  1. 基礎介面與訂單/會員系統

    • 基礎介面：開發使用者介面 (UI/UX)，包含首頁、商品列表、商品詳情頁等。| (7)
    • 會員系統：開發企業客戶的註冊、登入、帳戶管理、歷史訂單查詢功能。| (10)
    • 訂單系統：建立完整的下單與結帳流程。| (7)

  2. AI 禮品推薦 (流程 1)

    • 開發自然語言輸入介面 (e.g., 聊天視窗或搜尋框)。 | (3)
    • 開發 API 接收客戶需求，並實作核心流程： | (20)
      • 串接 **MongoDB** 查詢商品主檔。
      • 串接 **BigQuery** 查詢產品標籤與媒合數據 (可能涉及向量搜尋)。
      • 串接 **Gemini LLM (AI Agent)** 進行語意理解與推薦排序。
    • 將 AI 生成的推薦結果回傳並呈現在 UI 上。 | (1)

  3. AI 禮品打樣 (流程 2)

    • 開發介面，讓客戶能上傳 Logo 素材或輸入設計理念。 | (3)
    • 串接 Giftpack AI API，傳送產品與客製化需求，並接收生成的打樣預覽圖。 | (4)
    • 開發訂單確認流程，將客戶確認的打樣圖資料寫入MongoDB。 | (2)

B. 營運人員後台 (Operation-Facing Web App)

  1. 基礎介面與核心 CRUD 功能

    • 基礎介面：開發後台管理系統 (CMS) 的 UI/UX。 | (5)
    • 商品管理：** 包含商品主檔的上稿、編輯、下架 (CRUD)。 | (5)
    • 訂單管理：** 檢視、管理、更新前台傳來的訂單狀態。 | (5)
    • 客戶管理：** 管理企業客戶資料。 | (5)
    • 權限管理：** 系統登入登出、帳號管理。 | (5)

  2. AI 輔助 - 文案生成 (流程 1)

    • 在商品管理介面中，開發觸發 AI 生成文案的功能 (e.g., 傳送產品圖片/規格)。 | (3)
    • 串接 Gemini LLM (AI Agent) API 來生成文案。 | (3)
    • 將 AI 回傳的文案草稿存入 MongoDB。 | (1)

  3. AI 輔助 - 提案簡報生成 (流程 2)

    • 在訂單或客戶管理介面，開發觸發功能，並撈取MongoDB的訂單/產品資料。 | (3)
    • 串接 Gemini LLM (AI Agent) API，傳送資料以生成簡報文字內容。 | (3)
    • 接收 AI 回傳的簡報結構或內容。 | (1)

C. AI 核心與自動化排程 (AI Agent & Automation)

  1. Gemini LLM 串接服務層

    • 建立一個統一的 AI Agent 服務層 (作為後台服務)，集中管理所有對Gemini API的呼叫 (包含推薦、文案、貼標、媒合)。 | (3)

  2. 排程 G1 (商品貼標)

    • 開發一個自動化排程工作 (e.g., Cron Job / Azure Function Timer)。 | (2)
    • 流程：撈取MongoDB中未貼標的產品呼叫AI Agen將標籤結果回存至BigQuery。 | (2)

  3. 排程 G2 (行銷受眾包媒合)

    • 開發另一個自動化排程工作。
    • 流程：撈取MongoDB(產品) 與BigQuery(客戶資料/標籤)呼叫 AI Agent 進行媒合將生成的「受眾包」結果回存至BigQuery。 | (2)

D. 數據與基礎設施 (Data & DevOps)

  1. 資料庫設計 (Schema)

    • **MongoDB Schema:** 設計 Product, User, Order, AI_Draft 等的 Schema (資料結構)。
    • **BigQuery Schema:** 設計 Product\_Tags, Customer\_Segments, Audience\_Packages, Usage\_Logs 等的 Schema。

  2. 雲端建置 (DevOps)

    • 設定與配置兩台 Azure App Service (前台 Web App, 後台 Web App)。
    • 設定 **MongoDB (e.g., Atlas)** 的連線與安全配置。
    • 設定 Google BigQuery 的專案與權限。
    • (建議) 建立 CI/CD (持續整合/持續部署) 管線，自動化部署更新。
