讓我們的產品成為 AI 的首選工具

簡報目的：

  • 確保我們的 API 更容易被 Gemini、OpenAI(ChatGPT & Copilot) 等 AI 模型找到。
  • 確保 AI 能理解並交付正確且有效的服務資訊。
  • 提前佈局，搶佔 AI 應用生態系統的市場份額。

🚀 為了達成以上目的，我們找到了以下三個需要優化的項目

🚦 項目一：設置「AI爬蟲規則」 (robots.txt 配置)

robots是我們網站的Sitemap和AI爬蟲規則，指引 AI 爬蟲高效訪問取回資訊。

技術名稱 非技術說明(比喻) 為什麼重要？
`robots.txt` 如交通警察引導爬蟲到正確位置擷取資料 確保AI爬蟲能直接找到正確路徑擷取到「對外公開」且「最重要」的產品或服務資訊，節省它們的時間。
關鍵配置 開綠燈給公開路徑 告訴 AI：「這些是公開的，請盡情使用！」，避免 AI 浪費資源爬取私人路徑。

結論：設定好規則，避免 AI 迷路，讓我們產品或服務的關鍵資訊能被有效發現。

robots.txt 檔案內容範例：

User-agent: *
# 允許所有 AI 模型和爬蟲存取特定的公開 API 路徑
Allow: /promotions
Allow: /api/public/fare/estimate
Allow: /.well-known/openapi.json
# 建議：如果您的 API 文件有專屬路徑，也請允許
# Allow: /docs
# 選擇性：如果有網站地圖，加入以提升索引效率
# Sitemap: https://您的產品域名/sitemap.xml

🏷️ 項目二：Schema.org規範的「結構化標記」( JSON-LD   & HTML Tag)

就像超市商品上的標準化標籤，讓 AI爬蟲一眼識別我們是產品的重點資訊。

技術名稱 非技術說明(比喻) 為什麼重要？
JSON-LD 統一產品標籤 AI爬蟲不用「猜測」我們的網頁是什麼，而是直接讀取標籤，立即確定當下這個頁面的「重點資訊」。
HTML Tag 像房屋平面圖，讓人一看就知道哪裡是客廳、廚房或門口 讓AI爬蟲能「看得懂」網頁每個區塊內容，不會搞混主題，也能更容易被找到、正確收錄在爬取結果裡。

結論：讓 AI 快速分類並理解我們的產品類型，大幅提升被 AI 應用列入清單的機會。

📘 項目三：發佈「OpenAPI 規格書」

這是很關鍵的一步，提供一份全世界 AI 都看得懂的統一格式說明書。

技術名稱 非技術說明(比喻) 為什麼重要？
OpenAPI 規格 AI爬蟲可讀的標準化格式 讓AI爬蟲自動理解我們提供的OpenAPI有哪些參數和功能，並能提供外部廠商或開發者調用。
關鍵路徑 `.well-known/openapi.json` 將說明書放在一個標準、固定且容易被發現的地址，是進入 AI 應用生態的入場券。
（目前 OpenAI 與 Gemini 等主流大型語言模型並不會主動去調用公開的 API 服務）

結論：將Restful API 從「網頁資訊」升級為「AI 可用OpenAPI工具」。

讓我們的服務在可被被AI爬蟲識別為「公開可用服務資訊」。

開放的OpenAPI規格 (標準說明書)

吸引具備 AI 技術能力的合作夥伴，主動將我們的 API 納入 MCP Tools 生態中整合使用，拓展產品應用機會。
